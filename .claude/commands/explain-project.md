# 目的

このコマンドはプロジェクトの概要・アーキテクチャ・主要ファイルを説明します。

---

以下の手順でプロジェクトを調査し、わかりやすく説明してください。

## 調査する内容

1. **プロジェクト概要**
   - `README.md` または `CLAUDE.md` を読んでプロジェクトの目的を把握する
   - 使用技術スタックを列挙する

2. **ディレクトリ構造**
   - トップレベルのディレクトリ一覧を確認する
   - 主要なディレクトリの役割を説明する

3. **アーキテクチャ**
   - 採用しているアーキテクチャパターンを特定する（クリーンアーキテクチャ、MVC など）
   - 依存の方向（例: Controller → Service → Repository）を図示する

4. **エントリーポイント**
   - アプリケーションの起動ファイルを特定する
   - 主要なルーティング・エンドポイントを列挙する

5. **コーディング規約**
   - `CLAUDE.md` や `rules/` 配下のルールファイルから規約を読み取る
   - チームが守るべき制約事項を箇条書きにする

## 出力フォーマット

調査結果を以下の形式でまとめてください：

```
## プロジェクト概要
（目的と技術スタックを 3〜5 行で）

## ディレクトリ構造
（木構造で主要ディレクトリのみ、各行に役割コメント）

## アーキテクチャ図
（テキストで依存の方向を矢印で表現）

## 主要ファイル
（ファイルパス: 役割の一覧）

## チーム規約サマリ
（守るべきルールを箇条書き）
```

$ARGUMENTS が指定された場合は、その観点（例: "セキュリティ", "テスト方針"）を重点的に説明してください。
